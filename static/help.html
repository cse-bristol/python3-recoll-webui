What's this?

This is a simple search engine for the S-drive. It's based on a program called recoll.

The search index is updated every day at midnight, so anything from the previous day should come up.

This also means that the index will contain things that have been deleted until the next day.

Search syntax

Searching in files

If you write text in the box, it looks for documents containing those words, in any order.

If you put text in double quotes ("like this") it looks for documents containing the quoted phrase.

Searching file names

If you include something like ext:pdf or ext:docx it restricts the results to PDF or Word files (or files that have the text as a suffix after a dot, really).

If you want to look for a file with something in the name, you can write fn:stuff.

Combining or excluding things

Search terms are all required to match the document by default.

You can ask an either/or question using OR (uppercase is important). For example if you want either PDFs or word files containing the word validation, your query should be validation (ext:pdf OR ext:docx)

You can negate a term (require that it doesn't match) by prefixing it with a dash. For example if you want to find any non-csv document containing the word candlestick, your query should be candlestick -ext:csv
